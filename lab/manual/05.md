## 1. Milestone

Finalize environment configuration, database setup scripts, and README documentation to make the project reproducible and deployment-ready.

---

## 2. Key Requirements Introduced

- Environment variables must be externalized via `.env.local`
- `.env.local` must not be committed; `.env.example` must be provided
- Database setup must be split into:
  - user/db creation (`db-create.sh`)
  - schema initialization (`db-init.sh`)
- README must enable a new user to run the project end-to-end
- Remove unnecessary fields (`etf`, `testIssue`) from ticker seed data
- Ensure scripts (e.g., ticker sync) generate data matching TypeScript types

---

## 3. Design Decisions

- **Environment Handling**
  - Use `.env.local` for runtime secrets
  - Use `.env.example` as a template for onboarding

- **Database Setup Separation**
  - Infra layer (`db-create.sh`): creates DB + user
  - Schema layer (`db-init.sh`): creates tables, indexes, triggers

- **Script Design**
  - `sync-tickers.mjs` is the single source of truth for ticker data generation
  - Generated artifacts (`tickers.ts`) must not be manually edited

- **Data Modeling**
  - `TickerEntry` simplified to minimal lookup structure:
    ```ts
    type TickerEntry = {
      ticker: string
      name: string
      exchange: string
    }
    ```

- **Security / Robustness**
  - Replace unsafe env loading:
    ```bash
    export $(grep ...)
    ```
    with:
    ```bash
    set -a
    source .env.local
    set +a
    ```

- **Workflow Context Design**
  - Metadata, exposure, and analysis artifacts passed through a shared workflow context
  - Exposure data structured into `current` vs `target` summaries

## 4. Iterations / Corrections

- Issue:
  - `tickers.ts` contained fields (`etf`, `testIssue`) not defined in `TickerEntry`
- Fix:
  - Removed fields in `sync-tickers.mjs` instead of manually editing output
- Result:
  - Type-safe generated data aligned with domain model

---

- Issue:
  - Unsafe `.env.local` parsing using `grep | xargs`
- Fix:
  - Switched to `set -a; source .env.local; set +a`
- Result:
  - Safe handling of env variables (supports special characters, avoids injection issues)

---

- Issue:
  - Potential creation of DB user with empty password
- Fix:
  - Added guard:
    ```bash
    if [ -z "$DB_PASSWORD" ]; then exit 1; fi
    ```
- Result:
  - Prevents insecure DB setup

---

- Issue:
  - README did not reflect actual DB setup flow
- Fix:
  - Explicit separation of `db-create.sh` and `db-init.sh` steps
- Result:
  - Accurate onboarding documentation

---

- Issue:
  - Confusion about hidden `.env.local` file visibility
- Fix:
  - Clarified use of:
    ```bash
    ls -la
    ```
- Result:
  - Correct understanding of Unix hidden files
