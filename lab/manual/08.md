# 003_bootstrap-and-data-pipeline-setup.md

## 1. Milestone

Establish a complete database + data ingestion + initialization pipeline, and transition from schema setup to the beginning of a factor scoring engine.

---

## 2. Key Requirements Introduced

- Separate responsibilities between:
  - DB schema creation
  - Data ingestion (tickers, S&P500)
  - Tag extraction/import
  - Factor definition import
- Avoid mixing concerns (e.g., no schema creation inside tag/factor pipelines)
- Use shell scripts (`.sh`) for orchestration and `.mjs` for logic
- Introduce a full bootstrap process for initial setup
- Include full market universe (`sync-tickers.mjs`) before S&P500 subset
- Ensure scripts are reusable and independently executable
- Improve logging (file paths, counts) for generated artifacts

---

## 3. Design Decisions

### Architecture

- Adopt layered pipeline structure:
  - **DB Layer** → schema creation (`db-init.sh`)
  - **Data Layer** → external data sync (`data-sync.sh`)
  - **Definition Layer** → tag + factor initialization
  - **Orchestration Layer** → `bootstrap.sh`

### Script Responsibilities

- `.sh` → orchestration
- `.mjs` → business logic / data processing
- `.sql` → schema definition

### Pipeline Separation

- `data-sync.sh`
  - `sync-tickers.mjs` (full universe)
  - `sync-sp500-constituents.mjs`
  - `sync-sp500-metadata.mjs`

- `tag-init.sh`
  - extract candidates
  - import into DB

- `factor-init.sh`
  - import factor definitions only

### Naming

- Replaced `app-init.sh` → `bootstrap.sh`
  - clearer representation of full system initialization

### Execution Order

1. DB schema
2. Full ticker universe sync
3. S&P500 sync + metadata
4. Tag extraction/import
5. Factor definition import

---

## 4. Iterations / Corrections

- Issue: `tag-init.sh` included schema creation
  - Fix: Removed SQL execution from tag-init
  - Result: Clean separation of schema vs data pipelines

- Issue: `factor-init.sh` also included schema creation
  - Fix: Removed SQL execution
  - Result: Consistent pipeline design

- Issue: `outputPath is not defined` error
  - Fix: Introduced `const outputPath` before logging
  - Result: Script execution stabilized

- Issue: Errors did not stop pipeline execution
  - Fix: Added `process.exit(1)` in `.mjs` scripts
  - Result: Proper failure propagation to shell (`set -e`)

- Issue: Redundant npm script (`sync-tickers`)
  - Fix: Removed from `package.json`
  - Result: Single orchestration entry via `.sh`

- Issue: Environment variables loaded twice (shell + node)
  - Fix: Accepted duplication (non-critical), standardized usage
  - Result: Stable but slightly noisy logs

- Issue: dotenvx verbose logs
  - Fix: Identified as non-critical (can be removed later)
  - Result: No functional impact
