## 1. Milestone

Refined the project for public readiness by restructuring README, clarifying bootstrap limitations, and aligning system architecture with current data pipeline and portfolio logic.

---

## 2. Key Requirements Introduced

- README must be **public-facing and onboarding-friendly**
- Include a **Quick Start section** for immediate execution
- Support `.env.sample` usage for environment setup
- Clarify that `bootstrap.sh` includes **external API-dependent steps**
- Avoid misleading “fully automatic setup” claims
- Defer UI demo (screenshots/GIFs) until analysis/DB is complete
- Future direction: replace API-heavy setup with **seeded database**

---

## 3. Design Decisions

### System Positioning
- Transition from “portfolio tool” → **portfolio reasoning system**
- Emphasize **data pipeline + workflow architecture**

### Bootstrap Strategy
- Keep `bootstrap.sh` as:
  - DB init
  - data sync
  - tag/factor init
- But redefine as **development pipeline**, not guaranteed reproducible setup

### Environment Setup
- Standardize `.env.sample → .env.local` workflow
- Avoid embedding secrets in README

### README Structure
- Two-layer structure:
  1. **Quick Start (top)**
  2. Detailed system explanation (bottom)

### Data Strategy (Forward Plan)
- Move from:
  - API-driven metadata sync
- To:
  - **portable seeded DB / snapshot-based initialization**
