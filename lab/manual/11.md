---

## 1. Milestone

Finalize a production-ready SEC companyfacts bulk ingestion pipeline with idempotent execution, universe filtering, and state tracking, and transition toward growth factor computation.

---

## 2. Key Requirements Introduced

- Prevent redundant ingestion if archive is unchanged and already processed.
- Track both **archive state** and **ingest state** separately.
- Restrict ingestion to a defined **universe (S&P 500 tickers → CIKs)**.
- Reduce storage footprint (remove raw JSON, limit tags).
- Ensure ingestion is resumable and safe after interruption.
- Prepare data model for downstream **growth/factor computation**.
- Add progress logging and debugging visibility.

---

## 3. Design Decisions

### Data Models
- `sec_companyfact_raw`
  - Acts as **core fact table** (not full raw anymore).
  - Unique constraint on `(cik, taxonomy, tag, unit, accn, start, end)`.
  - Removed `raw_point_json` to reduce storage.

- `sec_companyfact_company_state`
  - Tracks per-company ingest metadata (activity, last processed, etc.).

- `sec_bulk_ingest_state`
  - Tracks archive + ingest lifecycle.
  - Single-row per dataset (not time-series).

---

### Abstractions
- Separation of concerns:
  - `ensureFreshSecBulkArchive` → archive freshness
  - `downloadSecBulkArchiveToFile` → download
  - `scanCompanyFactsZipEntries` → filtering
  - `processCompanyFactsZipEntries` → iteration
  - `flattenCompanyFacts` → transformation
  - `upsert*` → persistence

- Introduced **universe mapping layer**:
  - `mapTickersToCiks`
  - decouples ticker universe from ingest logic

---

### System Boundaries
- Ingest pipeline = **archive → scan → process → persist → state update**
- Growth layer explicitly separated (not mixed into ingest)

---

### Sequencing of Work
1. Archive freshness check
2. Conditional download
3. Load existing state
4. Universe filtering (CIK set)
5. Scan zip entries (skip unchanged)
6. Process documents
7. Persist facts + company state
8. Update ingest state
9. Skip future runs if unchanged

## 4. Iterations / Corrections

### Issue:
Ingest re-ran even when archive unchanged  
**Fix:** Removed `archive_mtime_iso` comparison and relied on `archive_file_size`  
**Result:** Deterministic archive identity, stable skip behavior

---

### Issue:
`archive_file_size` comparison failing  
**Fix:** Cast DB value to `Number(...)` before comparison  
**Result:** Correct equality check, skip condition works

---

### Issue:
`ingest_status` always reset to `idle`  
**Fix:** Changed `ON CONFLICT DO UPDATE` to preserve existing value when input is null  
**Result:** Proper lifecycle tracking (`running → completed`)

---

### Issue:
Missing progress tracking  
**Fix:** Added `currentIndex/total` with `process.stdout.write`  
**Result:** Real-time progress display without log spam

---

### Issue:
Incorrect CIK formatting (missing leading zeros)  
**Fix:** Normalize CIK using `padStart(10, "0")`  
**Result:** Consistent keying across tables

---

### Issue:
Excessive data volume (~6–8GB)  
**Fix:**  
- Removed `raw_point_json`  
- Introduced tag filtering  
- Limited universe  
**Result:** Manageable storage footprint

---

### Issue:
Unnecessary full-universe ingestion  
**Fix:** Added S&P500-based CIK filtering in scan phase  
**Result:** Reduced processing from ~20k → ~500 entities

- Identified need for **universe filtering** early (performance + cost)
- Challenged unnecessary ingestion expansion (submissions vs growth)
- Decided to:
  - drop raw JSON column
  - avoid over-engineering table relationships
- Directed architecture toward:
  - ingestion → factor pipeline separation
- Pushed for debugging via logs instead of assumption
- Chose **growth layer** over additional ingestion complexity
