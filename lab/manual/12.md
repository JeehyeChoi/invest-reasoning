# 003_metric-framework-and-ticker-chart.md

## 1. Milestone

Implemented the first end-to-end financial metric pipeline (revenue), including normalization, factor computation, API exposure, and UI chart rendering on the ticker detail page.

---

## 2. Key Requirements Introduced

- Build a full pipeline from SEC raw data → normalized series → chart visualization
- Ensure quarterly data consistency (handle YTD → quarterly conversion)
- Avoid artificial data filling for missing periods
- Support reusable structure for adding new metrics (net_income, cash flow, etc.)
- Maintain unique keys in UI rendering (React list stability)
- Keep system extensible via metric-specific normalization logic

---

## 3. Design Decisions

### Data Model

- Introduced layered types:
```ts
type Row = {
  end: Date | string;
  filed: Date | string | null;
  val: number | string;
  period_type: string;
  display_frame: string | null;
};
```

```ts
type SeriesPoint = {
  end: string;
  filed: string | null;
  val: number;
  periodType?: string;
  frame?: string | null;
};
```

```ts
type NormalizedSeriesPoint = {
  end: string;
  filed: string | null;
  val: number;
};
```

---

### Abstractions

- Introduced **metric-based normalization abstraction**:
```ts
type MetricNormalizer = (series: SeriesPoint[]) => NormalizedSeriesPoint[];
```

- Resolver pattern:
```ts
resolveMetricNormalizer(metricKey)
```

→ allows metric-specific logic

---

### System Boundaries

- Separation of concerns:

```text
SEC raw data → normalization → computation → API → UI
```

- Backend handles:
  - normalization
  - computation

- Frontend handles:
  - rendering
  - interaction (chart)

---

### Sequencing of Work

1. Build revenue normalization
2. Connect to factor computation
3. Expose via API
4. Render in ticker detail chart
5. Refactor into reusable metric framework

## 4. Iterations / Corrections

---

  - Fix: Avoid undefined variable or pass identifier explicitly
  - Result: Parser stabilized

---

- Issue: Duplicate filing items in UI
  - Fix: Recognized duplication originates from parsing layer
  - Result: Identified need for deduplication logic

---

- Issue: Missing data for GOOGL

  - Result: Identified need for ticker mapping layer

---

  - Fix: Introduced yearly tick builder + hover-based date display
  - Result: Improved chart readability without data distortion
